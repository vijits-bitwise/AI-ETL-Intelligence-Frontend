# Spec Files

Each file in this directory is a formal specification generated by `/analyze` before any code is written.

**Naming convention:** `<github-issue-number>-<kebab-slug>.md`

**Examples:**
- `05-add-developer-fields-card.md`
- `06-show-confluence-links-as-badges.md`
- `07-add-confidence-colour-threshold.md`

## Spec Lifecycle

| Status | Meaning |
|--------|---------|
| `DRAFT` | Generated by `/analyze`, not yet reviewed |
| `APPROVED` | Human-reviewed and committed — safe to run `/implement` |
| `IMPLEMENTED` | PR merged, issue closed |

## Workflow

```
/analyze "<requirement>"
    → generates spec → human approves
    → creates GitHub issue (gh issue create)
    → commits spec to main as docs/specs/<N>-<slug>.md

/implement "docs/specs/N-slug.md"
    → creates branch speckit/<N>-<slug> from main
    → implements exactly Section I files
    → outputs self-review checklist + PR description block

/review "docs/specs/N-slug.md"
    → validates implementation against spec (PASS/FAIL per section)
    → human creates PR from speckit/<N>-<slug> → main
```

## Commands

| Command | Pass | When to run |
|---------|------|-------------|
| `/analyze "<requirement>"` | Architect | For any new request — generates spec AND creates GitHub issue |
| `/implement "docs/specs/N-slug.md"` | Implementer | After spec is APPROVED — creates branch automatically |
| `/review "docs/specs/N-slug.md"` | Reviewer | After implementation, before PR |
| `/generate-tests "docs/specs/N-slug.md"` | — | Anytime after spec is APPROVED |

## Spec Sections

| Section | Content |
|---------|---------|
| A | Metadata (spec_id, issue_ref, change_type, status, components, api_spec_change) |
| B | Problem Statement (verbatim from issue) |
| C | API Contract Change (api-spec.yaml diff + TypeScript interface diff) |
| D | TypeScript Interface Changes (utils/api.ts) |
| E | Component Specification (components/ changes) |
| F | Page / Route Changes (app/ changes) |
| G | GITHUB-SPECKIT.md Updates |
| H | Test Cases (Given/When/Then) |
| I | Files Changed (table) |
| J | Out of Scope (verbatim from issue) |
